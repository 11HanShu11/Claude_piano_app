# おんぷノート 仕様書

ピアノ初学者向け練習手帳アプリ「おんぷノート」の機能仕様書。

- 文書バージョン: 1.1
- 対象アプリ: おんぷノート（単一HTMLファイル / PWA）
- 想定端末: iPhone 12 mini（横375px）を基準とするスマートフォン

> **1.1での主な追加**（読譜の「上達」と「継続」を強化）
> - 音階読み（白鍵）の回答を**固定ドレミパッド**化し、**1問ごとの反応時間**を計測
> - **弱点重み付け出題**と**適応モード（段階開放）**を追加
> - **スピードテスト**（読譜速度の定点測定）と**速度トレンドのグラフ**を追加
> - **音列読み**（並んだ音符を順に読む文脈読み）を追加
> - **連続記録（ストリーク）**と**デイリー目標**を追加

---

## 1. 概要

### 1.1 目的
ピアノ初学者が、音符の読み方・リズム・調号・音感を、スマートフォン1台で日常的に練習できるようにする。とくに**読譜（楽譜をスラスラ読む力）の上達**を、反応速度の可視化と弱点への集中で支援し、あわせて練習の記録・習慣化で継続を後押しする。

### 1.2 特徴
- インストール不要のWebアプリ（PWA）。iPhoneのSafariで開き「ホーム画面に追加」でアプリのように使える。
- サーバー・アカウント登録・課金は一切不要。
- 練習記録・日記・学習データは端末内（ブラウザのlocalStorage）にのみ保存され、外部送信されない。

### 1.3 非対象（スコープ外）
- 複数端末間でのデータ同期
- ユーザーアカウント／複数人での記録分離
- App Store等でのネイティブアプリ配布
- 実際の演奏（両手・運指）そのものの評価
- リマインド通知（iOSのWeb Push制約のため当面見送り）

## 2. 全体構成

### 2.1 画面（タブ）
下部のタブバーで4画面を切り替える。

| タブ | 役割 |
|------|------|
| 練習 | 各種練習問題・スピードテスト（メニューから選択して開始） |
| メトロノーム | テンポ練習用メトロノーム |
| 録画 | 練習の様子をカメラ録画・ダウンロード |
| 日記 | カレンダー・練習記録・日記・読譜スピードの記録・連続記録 |

### 2.2 技術構成
- 素のHTML / CSS / JavaScript による単一ファイル構成（フレームワーク不使用）。
- 五線譜・鍵盤・音符・調号・グラフの描画はSVG。
- 音の生成はWeb Audio API（音声ファイルは同梱しない）。
- データ永続化はブラウザのlocalStorage。

## 3. 練習機能

練習タブを開くとカテゴリ選択メニューが表示され、ボタンを選ぶとそのクイズが開始する。「← 選び直す」でメニューに戻る。カテゴリの練習では画面下部に「今日の正答数／問題数／経過時間」と「デイリー目標の進捗」を表示する。

### 3.1 カテゴリ一覧

| カテゴリ | 内部ID | 回答方法 | 音 | 記号切替 |
|----------|--------|----------|----|----------|
| 音階読み（白鍵） | note-white | 固定ドレミパッド | 正解音を再生 | ト音/ヘ音 |
| 音階読み（♯あり） | note-sharp | ドレミ4択 | 正解音を再生 | ト音/ヘ音 |
| 音階読み（鍵盤で） | note-keyboard | 鍵盤タップ | 正解音を再生 | 自動切替 |
| 音列読み | note-seq | 固定ドレミパッド（順に） | 各音を再生 | ト音固定 |
| 音符の種類 | duration | 4択 | なし | － |
| 調・調号 | key | 4択 | なし | － |
| 耳で音当て | ear-note | ドレミ4択 | 出題音を再生 | － |
| 和音の響き | ear-chord | 2択 | 和音を再生 | － |
| スピードテスト | （テスト） | 固定ドレミパッド | なし | ト音固定 |

### 3.2 音階読み（白鍵）
- 五線譜に音符を1つ表示し、**画面下部に固定配置した7つのドレミボタン（ド・レ・ミ・ファ・ソ・ラ・シ）**で答える。ボタンの位置は常に同じで、選択肢を探す時間や当てずっぽうを排除し、反応時間が読譜そのものを表すようにしている。
- ト音記号／ヘ音記号を切り替え可能。出題対象は白鍵のみ。
- 正解すると音符の音が鳴り、表示専用の鍵盤で該当位置が光る。不正解時は正しいボタンも光らせる。
- **1問ごとに反応時間を計測**し、音名別の学習データ（後述）に記録する。
- 音域の選び方は「適応／基本／ひろめ」から選べる（既定は「適応」）。

#### 3.2.1 出題音域モード
| モード | 内容 |
|--------|------|
| 適応 | 入口をドレミの3音に絞り、開放中の音がすべて習得済みになると1音ずつ開放する（ド→レ→ミ→ファ→ソ→ラ→シの順、最大7音）。読みやすい中央オクターブ（ト音 E4〜D5／ヘ音 C3〜B3）を用いる。 |
| 基本 | 従来の基本音域（ト音 E4〜C5／ヘ音 C3〜A3）から出題。 |
| ひろめ | 加線を含む広い音域から出題。 |

- 「習得済み」の判定：その音を4回以上出題し、誤答率25%以下。

#### 3.2.2 弱点重み付け出題
- どのモードでも、白鍵の音符は**音名ごとの弱点に応じた重み付け**で選ばれる（完全ランダムではない）。
- 重みは「未習・経験が浅い」「誤答率が高い」「反応が遅い」音ほど大きくなり、苦手な音が多めに出題される。
- 練習画面下部に、適応の進捗（例:「適応：ドレミ（3/7音）」）と苦手な音（例:「苦手：ソ ファ」）をヒント表示する。

### 3.3 音階読み（♯あり）
- 3.2と同様に五線譜へ音符を表示するが、回答は従来の**ドレミ4択**。
- 白鍵に加えて黒鍵の音（ド♯・レ♯・ファ♯・ソ♯・ラ♯）を出題する。該当する音のとき約50%の確率で♯付きの出題になる。
- 五線譜には音符の左に♯を描画。異名同音（♭表記）は用いず♯表記に統一する。
- ※固定パッド・適応・反応時間計測は白鍵（3.2）のみに適用。

### 3.4 音階読み（鍵盤で）
- 五線譜に表示された音符に対応する鍵盤のキーを、画面下のピアノ鍵盤をタップして答える。白鍵・黒鍵の両方が出題対象。
- 鍵盤は白鍵9個を表示。「◀ 低い音 / 高い音 ▶」で表示音域を1オクターブ単位で移動する（現在の音域はラベル表示）。
- 出題は常に「現在表示中の音域内」からのみ行う。全体の音域はC3〜C5。C4未満はヘ音記号、C4以上はト音記号を自動選択する。
- 回答すると押した鍵が色づき（正解=緑／不正解=赤）、正しい音が鳴る。不正解時は正解の鍵も光らせる。

### 3.5 音列読み（文脈読み）
- 五線譜に**4つの音符を横並び**で表示し、**固定ドレミパッドで左から順に**読んで答える。
- 出題範囲はト音記号 E4〜D5。**順次進行（±1音）を中心に、たまに跳躍（±2音）**する短い音列を生成し、実際のフレーズに近い読みを練習する。
- 現在読むべき音は五線譜上で強調表示（色＋▲マーカー）。回答した音は正解=緑／不正解=赤で着色する。
- 各音を答えるごとに正解音が鳴り、**1音ごとに反応時間を計測**して音名別の学習データ（3.2.2と共通）へ反映する。
- 1列を読み終えると正解数を表示し、次の列へ進む。

### 3.6 音符の種類
- 五線譜に音符を1つ表示し、全音符・二分音符・四分音符・八分音符の4択で答える。
- 符頭の白抜き／塗りつぶし、符尾の有無、旗の有無で描き分ける。

### 3.7 調・調号
- 五線譜に調号を表示し、何長調かを4択で答える。
- 収録: ハ長調・ト長調・ニ長調・イ長調・ヘ長調・変ロ長調・変ホ長調。

### 3.8 耳で音当て（絶対音感の入り口）
- 出題時に音が1つ自動再生され、ドレミの4択で答える。出題範囲は1オクターブ（ド〜シ）の白鍵。
- 「▶ もう一度きく」で再生し直せる。正解時に音を再度鳴らして確認できる。

### 3.9 和音の響き（相対音感の入り口）
- 和音が再生され、「明るい響き（メジャー）」「暗い響き（マイナー）」の2択で答える。
- ルート音はC4・D4・F4・G4・A4のいずれか。「もう一度きく」で再生し直せる。

### 3.10 スピードテスト（読譜速度の定点測定）
- 練習メニューから起動する、読譜速度を比較可能な形で測るベンチマーク。
- **固定分布＋毎回ランダム出題**：ト音記号 C4〜C5 の全7音名を均等に（各3回＝計21問）、順番だけ毎回ランダムに出題する。これにより日をまたいでも条件が揃い、純粋な速度の変化を測れる。
- 固定ドレミパッドで回答。上部に進捗バーを表示する。
- 終了後、**平均反応時間（秒／問。正答分のみ）**と**正答率**を表示し、前回テストとの差（例:「前回より0.30秒 速くなりました！」）を示す。
- 結果は練習ログとは別に保存し（`onpu:tests`）、日記タブのグラフの元データになる。テストを行った日はカレンダーの花丸対象になる。
- ※測定汚染を避けるため、テスト結果は練習ログ（弱点データ・日次集計）とは分離している。

## 4. メトロノーム

- テンポ範囲: BPM 40〜208（スライダー操作）。
- 拍子: 2拍子・3拍子・4拍子から選択。1拍目にアクセント音。
- 「タップでテンポ」ボタンを連打した間隔からBPMを自動算出。
- 発音はWeb Audio APIのルックアヘッド方式で、正確なタイミングを維持する。視覚インジケーター（点）が拍に同期して点灯する。

---

## 5. 録画

- カメラ・マイクを起動し、練習の様子を録画してファイル（webm形式）としてダウンロードできる。
- 録画データはアプリ内には保存せず、端末側にダウンロードする方式。
- カメラ・マイクへのアクセスは https 配信のページでのみ正常動作する。初回はブラウザの許可が必要。
- 録画を行った日は、日記の記録に「録画あり」として反映される。

## 6. 日記・カレンダー

### 6.1 カレンダー
- 月表示。練習した日に花丸「✿」を表示する。
- 「練習した日」の判定条件: その日に1問以上解いた、**スピードテストを受けた**、曲を記録した、または日記を記入した、のいずれか。
- 前後の月へ移動可能。当日は枠で強調。日付をタップするとその日が選択され、下の「選択中の日」枠に内容を表示する。

### 6.2 選択中の日（記録の閲覧・編集）
- カレンダーで選んだ日の記録を表示し、その場で編集・保存できる（過去日も編集可）。
- 記録項目:
  - 練習問題の正答数（カテゴリ別内訳＋合計、自動記録）
  - スピードテストの結果（最速タイム、自動記録）
  - 録画の有無（自動記録）
  - 今日練習した曲（複数入力可・自由入力）
  - 自由記述の日記

### 6.3 練習記録（正答数）の仕様
- 正答数はカテゴリ別に「正解数／問題数」を記録し、合計も表示する。
- 記録は日付ごとに独立する。日付をまたいでも他日の記録に加算されない。
- 同一日であれば、アプリを複数回開き直しても記録は上書きされず加算される。
- 日付判定は端末のローカルタイム基準（UTCによる日付ずれを起こさない）。
- 日記画面を開いた際に日付が変わっていた場合、選択日とカレンダーを自動的に当日へ戻す。

### 6.4 読譜スピードの記録（トレンドグラフ）
- スピードテストの結果を時系列で折れ線グラフ表示する（直近20回）。
- 縦軸は平均反応時間で、**速いほど上**に表示する。下に直近の履歴（日付・秒／問・正答率）を並べる。
- テスト記録がない場合は、テスト受講を促すメッセージを表示する。

### 6.5 連続記録・デイリー目標
- **連続記録（ストリーク）**：今日から遡って「練習した日」が連続した日数を表示する。今日がまだ未達でも、前日までの連続は途切れさせない。
- **デイリー目標**：1日あたりの目標問題数（既定20問／日、5〜100問を5問刻みで変更可）。日記タブで達成状況を、練習画面で当日の進捗（バー）を表示し、達成時は「今日の目標達成！🌸」を表示する。

## 7. データ保存仕様

### 7.1 保存先とキー
すべてブラウザのlocalStorageに保存する。

| キー | 内容 |
|------|------|
| `diary:YYYY-MM-DD` | 1日1レコードの練習記録・日記 |
| `onpu:tests` | スピードテスト結果の配列 |
| `onpu:noteStats` | 音名別の学習データと適応モードの開放段階 |
| `onpu:settings` | 設定（デイリー目標など） |

### 7.2 日次レコード構造（`diary:YYYY-MM-DD`）

| フィールド | 型 | 内容 |
|-----------|----|------|
| date | 文字列 | 日付 (YYYY-MM-DD) |
| songs | 配列 | 練習した曲名のリスト |
| diary | 文字列 | 自由記述 |
| correct | 数値 | その日の正解数合計 |
| total | 数値 | その日の問題数合計 |
| seconds | 数値 | 練習時間（秒） |
| recorded | 真偽 | 録画を行ったか |
| tested | 真偽 | スピードテストを行ったか |
| byMode | オブジェクト | カテゴリ別の {correct, total} |

### 7.3 スピードテスト結果（`onpu:tests` の各要素）

| フィールド | 型 | 内容 |
|-----------|----|------|
| ts | 数値 | 実施時刻（epochミリ秒） |
| date | 文字列 | 実施日 (YYYY-MM-DD) |
| avgMs | 数値/null | 平均反応時間（正答分。正答0ならnull） |
| correct | 数値 | 正解数 |
| total | 数値 | 出題数（通常21） |
| acc | 数値 | 正答率（%） |

### 7.4 音名別の学習データ（`onpu:noteStats`）
- `level`: 適応モードの開放段階（3〜7、開放している音名数）。
- `letters`: 音名（C〜B）ごとの `{ n:出題回数, wrong:誤答回数, ms:直近の正答反応時間の配列（最大10件） }`。
- 白鍵の音階読み・音列読みの回答時に更新し、弱点重み付けと段階開放の判定に用いる。

### 7.5 設定（`onpu:settings`）
- `dailyGoal`: デイリー目標の問題数（既定20）。

### 7.6 保存に関する注意
- 記録は「同じ端末・同じブラウザ」でのみ保持される。別端末・別ブラウザからは参照できない。
- ブラウザのサイトデータ削除により記録は消去される。
- 端末・ブラウザを共用すると記録が混在する（利用者分離の機能はない）。

## 8. 音の生成仕様

- クイズの音・メトロノーム・和音はすべてWeb Audio APIでリアルタイム生成する（音声ファイル不使用）。
- 単音はtriangle波でピアノ風の減衰を付与。和音は複数の周波数を同時発音する。
- メジャー／マイナーの和音は、ルート音に対し半音単位 [0,4,7]（メジャー）／[0,3,7]（マイナー）で構成する。
- iPhoneではマナーモード・音量設定の影響を受ける。ブラウザ仕様により、最初の発音時に画面タップが必要な場合がある。

## 9. 設計上の制約・方針

- 「上達」の対象は**読譜（楽譜を速く正確に読む力）**とし、演奏そのものは範囲外とする。上達の主指標を**正答時の反応時間（速度）**、副指標を正答率に置く。
- 練習（弱点重み付け・適応）とスピードテスト（固定分布のベンチマーク）を役割分担させ、練習ログは弱点対策に、テスト結果は速度トレンドの表示に用いる。両者を分けることで、日々のグラフが練習配合の偏りに汚染されないようにする。
- 音階読み（白鍵）の回答を固定ドレミパッドにしたのは、iPhone 12 mini（375px）でも7ボタンを1列に収められ、選択肢の探索時間・当てずっぽうを排除して反応時間を読譜そのものの指標にするため。
- 適応モードは、完全な初心者が「すべての音が弱点」の状態で圧倒されないよう、入口を3音に絞って段階的に開放する。
- 音列読みの音域をト音記号 E4〜D5 に限定するのは、加線を抑え、複数音符を1つの五線譜に収めて読みやすくするため。
- ♯・♭は「音階読み（♯あり）」「音階読み（鍵盤で）」「調・調号」で扱う。固定パッド・弱点重み付け・適応・反応時間計測は白鍵中心とする。
- 絶対音感トレーニングは本来長期の訓練を要するため、「耳で音当て」は音に親しむ入り口として位置づける。

## 10. 今後の拡張候補

- インターバル（度数）当て：2音の間隔（2度・3度…）を答えるモード。
- 音列読み・スピードテストのヘ音記号対応。
- ♯あり・鍵盤モードへの反応時間計測／弱点重み付けの展開。
- 苦手な音・カテゴリのより詳しい集計表示。
- リズム打ち練習（メトロノームに合わせて叩く）。
